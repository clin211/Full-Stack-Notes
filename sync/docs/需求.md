我想使用已经开发好的服务端接口 http://localhost:5555/v1/system/posts 去创建，接口的参数如下：
```json
 {
    "title": "棒呆向下许多同样地大约赦免横穿呼果然鉴于",
    "content": "consectetur Excepteur in",
    "cover": "eu deserunt ea",
    "summary": "deserunt labore",
    "categoryID": 76,
    "postType": 1,
    "position": 98,
    "status": 2,
    "tags": [
        51,
        100,
        40
    ]
}
```

下面来分别解释一下：
- title： 也就是文章的标题（文件名）
- content：文章的内容（文件中的内容）
- cover：文章的封面（现在可以使用同一个封面：https://images.unsplash.com/photo-1555066931-4365d14bab8c?q=80&w=1470&auto=format&fit=crop&ixlib=rb-4.0.3）
- summary：文章的摘要（文件中的内容的前100个字符）
- categoryID：文章的分类ID（基于文件夹结构映射，下面会整理映射关系）
- postType：文章的类型（现在只有一种，固定值 1）
- position：文章的位置（基于文件创建时间排序）
- status：文章的状态（现在只有一种，固定值 2）
- tags：文章的标签（基于文件中的标签提取，下面会整理映射关系）
